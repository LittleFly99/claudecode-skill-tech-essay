# Tech Essay - 专业技术文章写作技能

这是一个帮助撰写高质量技术文章的技能，提供结构化的写作框架、风格指导和模板支持。

## 核心理念

**好文章 = 清晰的结构 + 准确的表达 + 实用的价值**

技术文章的核心是让读者获得价值，无论是学到新知识、解决实际问题，还是拓展技术视野。

---

## 文章类型识别

根据用户的需求，自动识别文章类型：

| 类型 | 关键词 | 核心目标 |
|------|--------|----------|
| **技术教程** | 教程、指南、入门、详解、原理 | 帮助读者学会某项技术 |
| **实战案例** | 实战、解决、修复、实现、踩坑 | 展示具体问题的解决方案 |
| **项目经验** | 复盘、总结、经验、心得、体会 | 分享项目中的经验教训 |
| **技术分析** | 分析、调研、对比、趋势、研究 | 深入分析技术方案 |

## 写作流程

### 第一步：明确目标

在开始写作前，通过提问明确：

1. **目标读者是谁？**（新手/有经验的开发者/架构师）
2. **读者能获得什么？**（学技能/解决问题/拓展视野）
3. **核心观点是什么？**（一句话概括文章主旨）

### 第二步：选择结构

根据文章类型选择合适的结构模板：

- 教程类 → 使用 `templates/tutorial.md`
- 案例类 → 使用 `templates/case-study.md`
- 经验类 → 使用 `templates/experience.md`
- 分析类 → 使用 `templates/analysis.md`

### 第三步：填充内容

遵循「总-分-总」原则：

1. **开头**：简明扼要，点明主题和价值
2. **主体**：层次分明，逻辑清晰
3. **结尾**：总结要点，升华主题

### 第四步：打磨细节

- 检查代码示例是否完整可运行
- 检查技术术语是否准确
- 检查逻辑是否连贯
- 检查是否有遗漏的知识点

## 写作风格指南

### 混合模式原则

根据文章主题和目标读者，灵活选择写作风格：

| 风格 | 适用场景 | 特点 |
|------|----------|------|
| **专业严谨** | 原理讲解、底层分析 | 术语准确、逻辑严密、数据支撑 |
| **代码驱动** | 实战教程、功能实现 | 代码先行、注释详尽、结果验证 |
| **故事叙述** | 项目复盘、经验分享 | 场景带入、问题展开、解决方案 |
| **对比分析** | 技术选型、方案调研 | 多维对比、数据说话、结论明确 |

### 通用原则

1. **开门见山**：第一段就说清楚文章讲什么、读者能获得什么
2. **层次分明**：用标题、列表、代码块组织内容
3. **代码为王**：代码示例要完整、可运行、有注释
4. **图文并茂**：复杂概念用图示说明
5. **结尾升华**：总结要点、展望未来、引导互动

## 质量检查清单

写完文章后，对照检查：

- [ ] 标题是否准确概括内容？
- [ ] 开头是否清晰说明了文章价值？
- [ ] 结构是否层次分明、逻辑连贯？
- [ ] 代码示例是否完整可运行？
- [ ] 技术术语是否使用准确？
- [ ] 是否有足够的图表辅助说明？
- [ ] 结尾是否有清晰的总结？
- [ ] 是否对读者有实际帮助？
