# 技术文章评审规范

本文档定义技术文章的评审流程和标准，确保文章质量达标后才能发布。

---

## 评审流程

### 流程图

```
初稿完成 → 自我评审 → 技术审核 → 内容润色 → 最终确认 → 发布
    ↓           ↓           ↓           ↓           ↓
  [检查清单]  [同行评审]  [专家审核]  [语言优化]  [用户确认]
```

### 各阶段说明

| 阶段 | 执行者 | 目标 | 产出 |
|------|--------|------|------|
| 自我评审 | 作者 | 发现明显问题 | 问题清单 |
| 技术审核 | 技术专家 | 确保技术准确性 | 审核报告 |
| 内容润色 | 编辑/作者 | 优化表达和格式 | 修订稿 |
| 最终确认 | 作者 | 确认可发布 | 发布版本 |

---

## 第一阶段：自我评审

作者完成初稿后，必须进行自我评审。

### 自评清单

#### 基础检查

- [ ] 标题是否准确且吸引人？（符合五大爆款公式）
- [ ] 字数是否达到预期？（一般 1500-5000 字）
- [ ] 是否有明显的错别字和语法错误？
- [ ] 标点符号是否规范？
- [ ] 段落是否过长？（单段不超过 5 行为宜）

#### 结构检查

- [ ] 是否有清晰的目录结构？
- [ ] 标题层级是否正确？（不超过四级）
- [ ] 开头是否说明了文章价值？
- [ ] 结尾是否有总结和互动引导？
- [ ] 各部分之间过渡是否自然？

#### 内容检查

- [ ] 核心观点是否清晰？
- [ ] 技术术语是否准确？
- [ ] 代码示例是否完整可运行？
- [ ] 是否有足够的图表辅助说明？
- [ ] 参考资料是否标注来源？

#### 价值检查

- [ ] 读者能从文章中获得什么？
- [ ] 内容是否解决了标题承诺的问题？
- [ ] 是否有独创性见解或实用经验？
- [ ] 是否有可复用的代码或资源？

## 第二阶段：技术审核

技术审核由熟悉相关技术的专家进行，确保文章的技术准确性。

### 审核维度

#### 1. 技术准确性（权重 40%）

- [ ] 概念定义是否准确？
- [ ] 原理说明是否正确？
- [ ] 代码实现是否有 bug？
- [ ] 最佳实践是否符合行业标准？
- [ ] 是否存在技术误导？

#### 2. 完整性（权重 25%）

- [ ] 是否遗漏关键知识点？
- [ ] 前置条件是否说明清楚？
- [ ] 边界情况是否考虑？
- [ ] 依赖环境是否说明？

#### 3. 实用性（权重 20%）

- [ ] 代码是否可直接运行？
- [ ] 示例是否贴近实际场景？
- [ ] 是否提供了可复用的资源？
- [ ] 是否有明确的操作步骤？

#### 4. 时效性（权重 15%）

- [ ] 技术版本是否最新？
- [ ] 是否有过时的内容？
- [ ] 是否说明了版本兼容性？

## 第三阶段：内容润色

在技术内容确认后，进行语言和格式的优化。

### 润色清单

#### 语言优化

- [ ] 删除冗余表达
- [ ] 统一术语用词
- [ ] 优化长句结构
- [ ] 增加过渡语句
- [ ] 检查语病和错别字

#### 格式优化

- [ ] 统一标题层级
- [ ] 优化代码格式和注释
- [ ] 检查表格对齐
- [ ] 确保图片清晰
- [ ] 检查链接有效性

#### 可读性优化

- [ ] 添加必要的背景说明
- [ ] 增加小标题分段
- [ ] 突出关键内容（加粗、引用块）
- [ ] 添加总结性语句
- [ ] 确保逻辑流畅

## 评审标准

### 评分标准

| 分数 | 等级 | 说明 | 处理建议 |
|------|------|------|----------|
| 90-100 | 优秀 | 技术准确、内容充实、表达流畅 | 直接发布 |
| 80-89 | 良好 | 整体质量不错，有小瑕疵 | 小修后发布 |
| 70-79 | 合格 | 内容基本完整，有改进空间 | 修改后再审 |
| 60-69 | 待改进 | 存在明显问题 | 需要大改 |
| <60 | 不合格 | 存在严重问题 | 重新撰写 |

### 一票否决项

出现以下情况，直接判定不合格：

- ❌ 存在严重技术错误
- ❌ 代码无法运行或有安全漏洞
- ❌ 存在抄袭或版权问题
- ❌ 存在敏感信息泄露
- ❌ 严重误导读者

## 评审工作流集成

在 Skill 中，评审流程应该在文章撰写完成后自动触发：

```
文章完成 → 触发评审流程 → 输出自评清单 → 等待用户反馈 → 进入下一阶段
```

### 自动输出

当用户完成文章初稿后，Skill 应自动：

1. 输出「自我评审检查清单」
2. 生成「自评报告模板」
3. 提示进入技术审核阶段
4. 记录评审状态

### 示例输出

```markdown
## 📝 文章评审流程启动

您的文章初稿已完成！现在进入评审阶段。

### 第一阶段：自我评审

请对照以下清单进行自检：

#### 基础检查
- [ ] 标题是否符合五大爆款公式？
- [ ] 字数是否达标？
- [ ] 是否有错别字？
...

#### 技术检查
- [ ] 代码是否可运行？
- [ ] 术语是否准确？
...

### 下一步

完成自评后，请回复：
- 「自评完成」→ 进入技术审核阶段
- 「需要修改」→ 说明需要修改的内容
```
