# 技术写作风格规范

本文档定义技术文章的写作风格规范，确保文章风格统一、专业。

---

## 语言风格

### 专业性原则

1. **术语准确**：使用行业标准术语
2. **表述严谨**：避免模糊、绝对化的表述
3. **数据支撑**：用数据说话，避免主观臆断

### 易读性原则

1. **简洁明了**：一句话表达一个意思
2. **避免长句**：超过 30 字的句子考虑拆分
3. **少用被动语态**：主动语态更清晰

### 示例对比

| 不推荐 | 推荐 |
|--------|------|
| 这个问题可以被解决 | 我们可以解决这个问题 |
| 非常非常非常重要的 | 关键的 |
| 点击那个按钮 | 点击「提交」按钮 |
| 稍微有点慢 | 响应时间约 2 秒 |

## 标点符号

### 中文标点

- 中文句子使用中文标点：，。！？；：
- 标点后不加空格
- 专有名词用「」包裹

### 英文和代码

- 英文单词、数字与中文之间加空格
- 代码使用行内代码块：`code`
- 文件路径使用行内代码块

### 示例

```
推荐使用 React 18 版本，性能提升约 30%。
配置文件位于 `config/database.yml`。
点击「保存」按钮后，系统会自动验证。
```

## 格式规范

### 标题层级

```markdown
# 一级标题（文章标题）

## 二级标题（主要章节）

### 三级标题（小节）

#### 四级标题（细节说明）
```

**注意**：一篇文章只有一个一级标题，层级不超过四级。

### 列表格式

#### 无序列表

```markdown
- 第一项
- 第二项
  - 子项一
  - 子项二
- 第三项
```

#### 有序列表

```markdown
1. 第一步
2. 第二步
   1. 细节一
   2. 细节二
3. 第三步
```

### 代码块

#### 代码块格式

````markdown
```语言名称
// 代码内容
```
````

#### 示例

````markdown
```javascript
function hello(name) {
  return `Hello, ${name}!`;
}

console.log(hello('World'));
// 输出：Hello, World!
```
````

### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据 | 数据 | 数据 |
```

### 引用块

```markdown
> 这是一段引用文字
> 可以多行
```

## 代码规范

### 代码风格

1. 遵循语言惯例
2. 变量命名有意义
3. 添加必要注释
4. 保持代码简洁

### 注释规范

```javascript
// 好的注释：解释为什么
// 使用 debounce 避免频繁请求，减轻服务器压力
const handleSearch = debounce(search, 300);

// 不好的注释：重复代码内容
// 调用 search 函数
search();
```

### 完整示例

```javascript
/**
 * 防抖函数
 * @param {Function} fn - 要执行的函数
 * @param {number} delay - 延迟时间（毫秒）
 * @returns {Function} 防抖后的函数
 */
function debounce(fn, delay) {
  let timer = null;

  return function (...args) {
    // 清除之前的定时器
    clearTimeout(timer);

    // 设置新的定时器
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

// 使用示例
const handleInput = debounce((value) => {
  console.log('搜索:', value);
}, 300);

handleInput('React'); // 300ms 后输出：搜索: React
```

## 图片规范

### 图片要求

1. **清晰度**：分辨率足够，不模糊
2. **大小**：单张不超过 500KB
3. **格式**：截图用 PNG，照片用 JPG
4. **命名**：使用有意义的英文命名

### 图片位置

- 图片与正文之间空一行
- 图片下方可添加说明文字

### 示例

```markdown
![React 组件生命周期](./images/react-lifecycle.png)
*图1：React 组件生命周期示意图*
```

- 避免使用「点击这里」等模糊表述

## 术语表

### 统一用词

| 统一用词 | 避免用词 |
|----------|----------|
| 数据库 | 数据存储 |
| 接口 | API 接口 |
| 函数 | 方法（类中除外） |
| 变量 | 变量名 |
| 参数 | 参数值 |

### 缩写规范

- 第一次出现时写全称，括号注明缩写
- 后续使用缩写

**示例**：

> 单页应用（Single Page Application，SPA）是一种 Web 应用架构。SPA 的特点是...
