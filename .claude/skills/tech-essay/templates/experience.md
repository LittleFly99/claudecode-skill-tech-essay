# 项目经验文章模板

本文档提供项目经验/复盘总结类文章的标准结构和写作指导。

---

## 适用场景

- 项目复盘
- 经验分享
- 踩坑总结
- 团队协作心得

---

## 文章结构

### 1. 标题

**公式**：[项目/场景] + [核心收获]

**示例**：

- 618 大促：我是如何扛住 10W QPS 的
- 从 0 到 1：我们是如何搭建前端监控系统的
- 那些年我们一起踩过的坑：微服务拆分实录

---

### 2. 开头（200-300字）

```markdown
## 写在前面

[背景介绍] 这是一个什么样的项目？为什么要做？

[核心收获] 通过这个项目，我学到了：

- 收获一
- 收获二
- 收获三

[适合人群] 本文适合：

- [人群1]
- [人群2]
```

## 写作要点

### 经验提炼

1. **具体场景**：不要泛泛而谈，要结合具体场景
2. **可复制性**：让读者能应用到自己的项目中
3. **正反结合**：既说做对了什么，也说犯了哪些错

### 叙事风格

1. **时间线清晰**：按时间或阶段组织内容
2. **有故事性**：适当加入场景描写
3. **有思考**：不仅是做了什么，还有为什么

### 价值导向

1. **解决问题**：经验要能解决实际问题
2. **避免重复错误**：让读者少走弯路
3. **可借鉴**：提供可落地的建议

## 质量检查

- [ ] 背景是否交代清楚？
- [ ] 过程是否完整回顾？
- [ ] 经验是否具体可落地？
- [ ] 教训是否有分析有解决方案？
- [ ] 是否有给读者的实际建议？
